📊 데이터 모델링의 핵심 개념

🔧 데이터 모델링의 3가지 구성요소

🎯 어떤 것 (Entity)

실체, 개체
예: 학생, 과목, 교수

🔗 연관성 (Relationship)

개체 간의 관계
예: 학생이 과목을 수강한다

🏷️ 성격 (Attribute)

개체의 속성
예: 이름, 나이, 학번

🏗️ 스키마 vs 인스턴스

📋 스키마 (Schema)

데이터의 구조와 규칙을 정의하는 틀

  • 데이터베이스의 설계도
  • 어떤 테이블이 있고, 어떤 컬럼들이 있는지
  • 데이터 타입, 제약조건 등을 정의
  • 변경이 어려우며, 안정적
📊 인스턴스 (Instance)

스키마에 따라 실제 저장된 데이터

  • 테이블에 들어있는 실제 레코드들
  • 구체적인 값들
  • 시간에 따라 계속 변화
  • 삽입, 수정, 삭제가 빈번함

📝 스키마와 인스턴스의 용어 대응

⚠️ 중요: 같은 개념, 다른 용어!

스키마와 인스턴스에서는 같은 개념을 서로 다른 용어로 표현합니다. 이것이 데이터 모델링을 이해하는 핵심입니다!

📊 스키마 vs 인스턴스 용어 대응표
구성요소 스키마 레벨 인스턴스 레벨 테이블 관점
🎯 어떤 것 엔터티 타입
(Entity Type)		 엔터티
(Entity)		 테이블 구조
엔터티
(Entity)		 인스턴스
(Instance/어커런스)		 테이블의 각 행
🔗 연관성 관계
(Relationship)		 페어링
(Pairing)		 관계 정의 ↔ 실제 연결된 데이터
🏷️ 성격 속성
(Attribute)		 속성값
(Attribute Value)		 컬럼명 ↔ 각 셀의 실제값

📋 구체적인 예시로 이해하기

📋 스키마 레벨
🎯 엔터티 타입: "학생"
→ 학생이라는 개념/테이블 자체
🎯 엔터티: "개별 학생 개체"
→ 테이블의 각 행이 될 개체
🔗 관계: "수강한다"
→ 학생과 과목 간의 관계 정의
🏷️ 속성: "이름", "학번", "나이"
→ 컬럼명들
📊 인스턴스 레벨
🎯 엔터티: "학생" 테이블
→ 실제 테이블 (엔터티 타입의 구현)
🎯 인스턴스: "김철수", "이영희"
→ 실제 학생 개인들 (각 행의 데이터)
🔗 페어링: "박민수-수강-수학I"
→ 실제 학생과 과목 간의 수강 연결
🏷️ 속성값: "김철수", "20230001", "20"
→ 각 셀의 실제 데이터

📚 실제 예시: 학생 관리 시스템

🎯 스키마 관점: "학생 테이블은 학번(정수), 이름(문자열), 나이(정수), 전공(문자열) 컬럼을 가진다"

스키마: 학생 테이블 구조 정의
컬럼명 데이터 타입 제약조건 설명
학번 INTEGER PRIMARY KEY, NOT NULL 학생 식별자
이름 VARCHAR(50) NOT NULL 학생 이름
나이 INTEGER CHECK (나이 > 0) 학생 나이
전공 VARCHAR(100) - 전공 분야

📊 인스턴스 관점: "실제로 저장된 학생들의 구체적인 정보"

인스턴스: 실제 저장된 학생 데이터
학번 이름 나이 전공
20230001 김철수 20 컴퓨터공학
20230002 이영희 19 경영학
20230003 박민수 21 수학과
20230004 최지은 20 물리학

💡 핵심 포인트

  • 스키마의 "엔터티 타입" = 인스턴스의 "엔터티" (테이블 개념)
  • 스키마의 "엔터티" = 인스턴스의 "인스턴스" (각 행의 실제 데이터)
  • 속성은 컬럼명, 속성값은 각 셀의 실제 데이터
  • 같은 개념이지만 관점에 따라 용어가 달라지는 것이 특징!

🔄 스키마와 인스턴스의 관계

스키마 (설계도) → 인스턴스 (실제 데이터)
집의 설계도 → 실제로 지어진 집
요리 레시피 → 실제로 만든 요리
클래스 정의 → 객체 인스턴스
